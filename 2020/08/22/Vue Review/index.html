什么是虚拟DOM？

什么是DOM？

Document Object Model (DOM) 就是将HTML看作树形结构的API，每个节点代表了HTML的一个对象。

DOM 允许我们通过Javascript编程的方式改变文档的结构，样式，内容。搜索和更新真正的DOM是Web应用性能的瓶颈，使用虚拟DOM就是为了更快。

M $\rightarrow$ V的过程

  1. 模版 template 编译成渲染函数 render（仅发生一次，可以在服务器端运行来提高性能）
  2. 执行渲染函数 render 来创建虚拟DOM节点 vnode，和旧的vode做对比，仅更新那些实际发生内容更改的（patching 算法，基于 Snabbdom 的实现）。（在虚拟DOM里面搜索和更新是很快的，我们将节省了任何不必要的真实的DOM操作）

Patching

  • 初次渲染的时候，将VDOM渲染成真正的DOM然后插入到容器中
  • 再次渲染的时候，将新旧vnode做diff算法，递归地找到同级间的diff，最终将差异应用到真正的DOM树上

防抖 VS 节流

防抖（debounce）和节流（throttle）都是在n秒内只能执行一次，区别在于，防抖是如果在n秒内还有执行的话，那么会重新开始算，

bind、call、apply的区别

函数运行时间 绑定时间
function object f 未来 未来
function call f() 现在 现在
f.call() 和 f.apply() 现在 现在
f.bind() 未来 现在

bind()允许现在设定好this之后迟一点再运行，只是连上/绑定。

call() 就是知道明确需要多少个参数去调用，就写多少个。apply()就是传一个数组应用进去。

组件

组件是可复用的 Vue 实例，所以它们与 new Vue 接收相同的选项，例如 data、computed、watch、methods 以及生命周期钩子等，仅有的例外是像 el 这样根实例特有的选项。

常见状态码

  • 200：成功
  • 301：网页已经永久移动到新位置，自动跳转到新位置
  • 302：临时移动，暂时从另一个位置响应，下次用原位置来请求就行
  • 403：禁止，服务器拒绝请求（你的IP被列入黑名单/过多地访问此网站(一般是用采集程序)，被防火墙拒绝访问了/以http方式访问需要ssl连接的网址/连接的用户过多，可以稍后再试）
  • 404：服务器找不到网页

CSS Box 模型

给content加上一个border，然后content到border的距离就是padding，border和另一个元素的border之间的距离就是margin。background-color就是border所框定的范围的颜色，也就是padding的范围内是会上色的。对于可点击的div的padding范围也都是可点击的。

inline 元素的 box 特性

  • margin：左右会生效，上下会失效
  • padding：上下左右都生效，但是上下的其他文字不会改变，只是本元素的范围扩大了（通过background-color可以看到覆盖了上下的文字）
  • 希望有正常的效果？display: inline-block;就行

margin collapse

上方元素的margin-bottom和下方元素的margin-top会取其中较大的那个作为两个元素间的间距。想要解决这个问题，可以在中间增加一个height:1px;的div

Box Sizing

padding是在原本的width基础上额外增加的宽度。使用box-sizing: border-box;就可以将width的宽度解释为以border为边界的box的宽度。

大小

  1. em：多少倍的字体
  2. rem：根据根节点的缩放
  3. vh：屏幕可见高度的1%，也就是说height:100vh == height:100%;，区别在于当元素没有内容的时候，后者不会被撑开，前者始终保持高度与屏幕大小一致
  4. vw：屏幕可见宽度的1%，width:100%是按照父元素的宽度，但是100vw是可见宽度，所以50vw可能会比50%更大

Flex container

flex-direction

预设值是row（从左到右），还有column（从上到下），row-reverse（从右到左）和column-reverse（从下到上）

main-axis 主轴 cross-axis 交叉轴
row row column
column column row

justify-content & align-items

  • justify-content是指靠近主轴的方向

  • align-items是指靠近交叉轴的方向

  • justify-content 和 align-item可以用的参数有 center（居中），flex-start（靠近flex-direction的起始方向），flex-end（靠近flex-direction的结束方向）

flex-wrap

flex-wrap的默认值是nowrap，无论有多少个项目也会挤在一行，改成wrap就会自动分行

flex-flow(flex-direction + flex-wrap)

例 flex-flow: column wrap;

align-content

如果有多于一行的flex items（即flex-wrap设为wrap）的时候，行与行之间的对齐方式。

Flex Item

order

默认是0，如果我们设定order:-1;那么会排在最前面，因为默认是按从小到大来排序的。如果设为order: 1;的话，那么会排在最后。可以随意将个别的flex item的位置进行调整。

align-self

用来覆写align-items的设定的。可以将某一个具体的flex item在交叉轴上的位置进行改变（比如原本是center，在y轴上居中，可以改成是flex-end/flex-start来使得它贴近底部或者顶部）

flex

flex-grow

在主轴上的剩余空间的加权。flex-grow默认为1，即所有的flex item会平分多余的空间。

  • 设定为0的话就不会大小不会变

flex-shrink

在主轴上的超出空间的分担加权，预设值默认为1，即所有的flex item会平均分担需要缩小的空间。

  • 设定为0的时候大小不会变。

flex-basis

flex item的主轴方向上的item大小（不改变交叉轴上的方向大小）使用了flex-basis之后原有的宽度设定值（当主轴是y轴的时候是高度）就会失效。

  • 设定为0的时候，大小就只取决于内容的大小。

  • 设定为auto的时候，就会按照本身的大小。

SetInterval vs SetTimeOut

首先这两者都是在运行完执行栈之后指定的毫秒数后调用，都很可能不是马上执行的。

不同

setInterval是被设定的函数的起始时间相差一个interval。setTimeout则是函数结束到函数开始间的delay。

假设说我们的函数需要运行80ms，使用setInterval 100ms来看的话，实际函数间的间隔只有20ms，但是setTimeout是不考虑函数运行时间长短的，只看实际delay。如果函数运行需要150ms，那么setInterval会在运行完成后检查间隔并马上再次运行。

嵌套setTimeout可以保证均匀的delay

零毫秒设置超时

在真实的浏览器中，因为很多脚本都有4ms的限制，所以浏览器中事实上会强制五个或更多嵌套调用的最小延迟为至少4ms。

Promise

https://andyyou.github.io/2017/06/27/js-promise/

Promise简单的说就是为了解决callback hell而产生的语法。因为then的连接不需要indent，非常容易看清楚。

就是先有一个promise，可能会花很长时间才能知道结果（如下载），结果可能会成功（resolve）或者失败（reject）。然后我们用then去接resolve返回的内容，用catch去接reject返回的内容。
