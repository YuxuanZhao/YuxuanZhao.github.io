Box

给content加上一个border，然后content到border的距离就是padding，border和另一个元素的border之间的距离就是margin。

background-color就是border所框定的范围的颜色，也就是padding的范围内是会上色的。对于可点击的div的padding范围也都是可点击的。

Box Sizing

padding是在原本的width基础上额外增加的宽度。使用box-sizing: border-box;就可以将width的宽度解释为以border为边界的box的宽度。

默认是box-sizing: conten-box;

IE将宽度解释为border-box

inline 元素的 box 特性

  • margin：左右会生效，上下会失效
  • padding：上下左右都生效，但是上下的其他文字不会改变，只是本元素的范围扩大了（通过background-color可以看到覆盖了上下的文字）
  • 希望有正常的效果？display: inline-block;就行

Margin Collapse

上方元素的margin-bottom和下方元素的margin-top会取其中较大的那个作为两个元素间的间距，也发生在父子之间。

解决方法：在发生了重合的元素外包裹一层容器，并且设置该容器overflow: hidden;

Flex布局

Flex container

flex-direction：

预设值是row（从左到右），还有column（从上到下），row-reverse（从右到左）和column-reverse（从下到上）

main-axis 主轴 cross-axis 交叉轴
row row column
column column row

flex-wrap：

flex-wrap的默认值是nowrap，无论有多少个项目也会挤在一行，改成wrap就会自动分行

flex-flow：

(flex-direction + flex-wrap)： flex-flow: column wrap;

justify-content：

justify-content是指靠近主轴的方向

align-items：

align-items是指靠近交叉轴的方向

justify-content 和 align-item 常用的参数有 center（居中），flex-start（靠近flex-direction的起始方向），flex-end（靠近flex-direction的结束方向）

align-content：

如果有多于一行的flex items（即flex-wrap设为wrap）的时候，行与行之间的对齐方式。

Flex Item

order：

默认是0，如果我们设定order:-1;那么会排在最前面，因为默认是按从小到大来排序的。如果设为order: 1;的话，那么会排在最后。可以随意将个别的flex item的位置进行调整。

align-self：

用来覆写align-items的设定的。可以将某一个具体的flex item在交叉轴上的位置进行改变（比如原本是center，在y轴上居中，可以改成是flex-end/flex-start来使得它贴近底部或者顶部）

flex：

flex-grow：

在主轴上的剩余空间的加权。flex-grow默认为1，即所有的flex item会平分多余的空间。

  • 设定为0的话就不会大小不会变

flex-shrink：

在主轴上的超出空间的分担加权，预设值默认为1，即所有的flex item会平均分担需要缩小的空间。

  • 设定为0的时候大小不会变。

flex-basis：

flex item的主轴方向上的item大小（不改变交叉轴上的方向大小）使用了flex-basis之后原有的宽度设定值（当主轴是y轴的时候是高度）就会失效。

  • 设定为0的时候，大小就只取决于内容的大小。

  • 设定为auto的时候，就会按照本身的大小。

位置

Float：

一般用于图片，使得文本围绕着图片周围，可以是left，right，none（不浮动），inherit（从父元素继承float的值）。使用了float之后默认display:block;

清除浮动

浮动带来的问题：

  1. 父元素的高度无法被撑开，影响与父元素同级的元素
  2. 与浮动元素同级的非浮动元素（内联元素）会跟随其后
  3. 若非第一个元素浮动，则该元素之前的元素也需要浮动，否则会影响页面显示的结构。

清除浮动的方式：

(1) 在子元素后添加一个空div div{clear:both;}

(2) 在父元素中{overflow:hidden|auto;zoom:1}

(3) :after伪选择符，在父容器的尾部自动创建一个子元素

"clearfix"是父容器的class名称

"content:"020";"是在父容器的结尾处放一个空白符，

"height: 0;"是让这个这个空白字符不显示出来

"display: block; clear: both;"是确保这个空白字符是非浮动的独立区块

:after选择符IE 6不支持,添加一条IE 6的独有命令"zoom:1;"就行了.

position：

默认：static + top/bottom/left/right是不会改变原本的位置的

绝对：absolute，对第一个父元素进行定位

固定：fixed，相对于浏览器窗口进行定位，滚动网页对它无效

相对：relative，相对其自身原本正常的位置进行定位

粘性：sticky，-webkit-sticky（safari）+top:0的话在滚动至元素上部贴着窗口上沿的时候会保持在上沿（类似于冻结了首行的excel）

置中

左右

  • inline元素text-align:center;
  • block元素margin: 0 auto;

上下

Position Absolute（置中元素本身）

潮流：Flexbox（置中元素内容，把元素大小调整到画面大小）

过时：Display Table（置中元素内容，把元素大小调整到画面大小）

大小

  1. em：多少倍的字体
  2. rem：根据根节点的缩放
  3. vh：屏幕可见高度的1%，也就是说height:100vh == height:100%;，区别在于当元素没有内容的时候，后者不会被撑开，前者始终保持高度与屏幕大小一致
  4. vw：屏幕可见宽度的1%，width:100%是按照父元素的宽度，但是100vw是可见宽度，所以50vw可能会比50%更大
  5. Chrome使用小于12px

宽度高度的百分数

用百分数设定宽度和高度的时候，都是按父容器的宽度！

响应式设计

响应式网站设计(Responsive Web design)是一个网站能够兼容多个终端，而不是为每一个终端做一个特定的版本。

基本原理是通过媒体查询检测不同的设备屏幕尺寸做处理。

页面头部必须有meta声明的viewport。

overflow:

参数是 auto 时候，子元素内容大于父元素时出现滚动条。
参数是 visible 时候，溢出的内容出现在父元素之外。
参数是 hidden 时候，溢出隐藏。

  • >：E > F 表示选择E元素的所有子F元素，与E F的区别在于bai，E F选择所有后代元素，>只选择一代。

  • +：E+F表示HTML中紧随E的F元素。

  • .；class

  • #：id

  • li a：空格是后代选择器

  • *：通配符

  • 属性选择器（a[rel = “external”]）

  • 伪类选择器（a:hover，li:nth-child，:enabled，:disabled 控制表单控件的禁用状态，:checked 单选框或复选框被选中）

  • 伪元素选择器（::before在元素之前添加内容，也可以用来做清除浮动，::after在元素之后添加内容）

display:

值 描述
inline 默认。此元素会被显示为内联元素，元素前后没有换行符
block 此元素将显示为块级元素，此元素前后会带有换行符

list-item 此元素会作为列表显示

默认inline-block的元素

<input> 、<img> 、<button> 、<textarea>

display:none vs visibility:hidden

display : none 隐藏对应的元素，在文档布局中不再分配空间（回流+重绘）

visibility:hidden 隐藏对应的元素，在文档布局中仍保留原来的空间（重绘）

使用 CSS display:none 属性后，HTML 元素（对象）的宽度、高度等各种属性值都将“丢失”;而使用 visibility:hidden 属性后，HTML元素（对象）仅仅是在视觉上看不见（完全透明），而它所占据的空间位置仍然存在。

rgba() vs opacity 的透明效果

opacity作用于元素，以及元素内的所有内容的透明度，

rgba()只作用于元素的颜色或其背景色。（设置rgba透明的元素的子元素不会继承透明效果！）
